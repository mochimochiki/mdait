# mdait 設計ドキュメントインデックス

このディレクトリは mdait の設計情報をまとめたものです。知りたい内容に応じて以下から該当するドキュメントを参照してください。

### 全体像を知りたい

**[design.md](design.md) - 全体設計書**
- **読むべき時**: mdait の全体像を把握したい、アーキテクチャを理解したい
- **内容**: システム概要、階層構造、mdaitUnit の仕組み、マーカー構造、ユニット境界の定義

### コマンドの動作を知りたい
**[commands.md](commands.md) - コマンド層設計**
- **読むべき時**: sync/trans/term などのコマンドの仕様を確認したい、処理フローを理解したい
- **内容**: setup/sync/trans/term、コマンド実行シーケンス図

### コア機能の実装を理解したい
**[core.md](core.md) - コア機能層設計**
- **読むべき時**: mdaitUnit の内部構造、ハッシュ計算、ステータス管理の仕組みを知りたい
- **内容**: mdaitUnit、Hashによる差分検出、Status管理、ユニット同期のシーケンス図

### UI の仕組みを知りたい
**[ui.md](ui.md) - UI層設計**
- **読むべき時**: ツリービュー、ホバー表示、CodeLens などの UI 実装を理解したい
- **内容**: StatusTreeProvider、翻訳サマリ表示、インライン表示、WelcomeView、UI更新シーケンス図

### 設定ファイルの構造を知りたい
**[config.md](config.md) - 設定管理層設計**
- **読むべき時**: mdait.json の構造、設定の読み込み方法を確認したい
- **内容**: mdait.json 設定項目一覧、Configロードのシーケンス図

### AI との連携を理解したい
**[api.md](api.md) - API層設計**
- **読むべき時**: AI サービスとの通信方法、プロバイダーの切り替え方法を知りたい
- **内容**: AIService インターフェース、AIServiceBuilder（プロバイダ選択）

### プロンプトの内容を確認したい
**[prompt.md](prompt.md) - プロンプト設計**
- **読むべき時**: AI に送信するプロンプトの仕様、入出力形式を確認したい
- **内容**: trans（翻訳）プロンプト、term-detect（用語検出）プロンプト、入出力の JSON フォーマット

### ユーティリティ機能を知りたい
**[utils.md](utils.md) - ユーティリティ層設計**
- **読むべき時**: ファイル探索などの汎用機能の実装を知りたい
- **内容**: FileExplorer（Markdown ファイル探索）

### テストの方針を知りたい
**[test.md](test.md) - テスト層設計**
- **読むべき時**: テストの書き方、実行方法を確認したい
- **内容**: テストカテゴリ、サンプルワークスペース、実行シーケンス図、テスト実践のプラクティス
