# wish list

## ガイドライン

**IMPORTANT**
このWishリストからタスクチケットを作成する場合は以下の手順に従うこと。

1. 該当のWishの内容を確認する（関係ないWishについて混同しないように注意）
2. `/design`ディレクトリ以下の設計書を確認し、ユーザーとともに意図や全体との整合性が取れるまで十分な設計を行う。
3. タスクチケットを発行する
4. 本リストからは該当Wishの章を削除する

## 要望一覧

### 品質ゲートの再考

現在の品質チェックの機能はとりあえずの実装であり精度がよくない。品質ゲートとして設けるチェック機能を再考する。
- 現在実装済みの機能は一度本当に必要か、制度が十分か棚卸しする
- どのような品質チェックが有効か？

### オンデマンド翻訳

mdait管理と別で、テキスト選択範囲のオンデマンド翻訳を行う機能を追加する。
- mdaitの思想にうまく適合させるにはどのような設計がよいか？
- UXの良いインターフェースはどのようなものか？
- テキスト選択範囲以外に翻訳対象を指定する方法はあるか？

### .mdait ディレクトリ作成時の.gitignore自動生成

.mdaitディレクトリ作成時、`.mdait/.gitignore`を生成し、logディレクトリを無視する設定を自動的に追加する。

### ステータスパネルと開いているファイルの同期

エディタで開いているファイルに応じてステータスパネルの開閉を自動的に切り替える機能を追加する。
- VSCodeのエクスプローラと同等の機能

### .md以外のファイルのサポート

.md以外のテキストファイル（例: .txt, .json, .yaml .csvなど）もサポートする。
- mdと違いhtmlコメントは使えないためファイル単位で扱う必要がある
- mdaitマーカーはどこに持つのか？検討する必要がある

### Statusアイコンの改善

- need:reviewやneed:reviseは視覚的にわかりやすいアイコンにする
- .mdファイル階層と、その下の見出し階層を見分けられるような構造検討

### frontmatterのサポート

mdait.xxxx という形式でfrontmatter(yaml)に情報を持たせることをサポートする。
- level : これまでautoMarkerLevelとしていたが、levelという名称にシンプル化し、frontmatterでも指定できるようにする
- frontmatter指定がある場合そちらが優先されるようにする

### frontmatterの翻訳サポート

frontmatter内の値も翻訳できるようにする。
- key: value 形式のうちvalue部分を翻訳する
- どのkeyを翻訳対象とするか設計する必要がある
- frontmatter自体のmdaitマーカーもfrontmatterに持たせ、`mdait.front`とする

### 横並びファイルのtransPairのサポート

現在mdaitはja/enのように言語ごとにディレクトリを分けて翻訳ペアを管理しているが、ja.md/en.mdのように横並びでファイルを配置するケースもある。これらをサポートする。
- 両立は難しそうなので、設定で切り替えられるようにする
