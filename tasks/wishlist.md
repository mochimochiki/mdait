# wish list

## ガイドライン

**IMPORTANT**
このWishリストからタスクチケットを作成する場合は以下の手順に従うこと。

1. 該当のWishの内容を確認する（関係ないWishについて混同しないように注意）
2. `/design`ディレクトリ以下の設計書を確認し、ユーザーとともに意図や全体との整合性が取れるまで十分な設計を行う。
3. タスクチケットを発行する
4. 本リストからは該当Wishの章を削除する

## 要望一覧

### frontmatterのlevel指定動作の改善

ja->enなどでfrontmatterのlevel指定がある場合の挙動がおかしい。そもそもlevel指定を各ファイルで行うべきかsrcのみで行うべきかなど。また、各言語ごとに指定された場合の動作仕様などが決まっていない

### conflict時の仕様改善

`need:solve-conflict`の時にどのように対処すればいいかが明確でない。次の操作補助が必要。また、conflict解消前に再度syncしたときの動作が不安定。
conflict発生後にsyncするとconflictが上下ユニットに伝搬してしまい冪等性が保たれない。

### 用語集翻訳のサポート

未翻訳時にまず用語集を整備したい。用語集自体の穴埋め翻訳があればよいはず

### TMのサポート

翻訳メモリ（TM: Translation Memory）をサポートする。
通常の翻訳メモリと違い、src側のハッシュと文章の紐づけさえできていればよい。近い文の含まれるユニットをLLMにプロンプトで参考として渡す。

### Target側のディレクトリの総ユニットカウント不具合

x/yのかたちで、x: 翻訳済みユニット数、y: 総ユニット数を表示しているが、yの数え方がおかしい。おそらくsrcが存在しないので翻訳不要のユニットまでカウントされている。

### 校正のサポート

すでに翻訳済みの翻訳ペアに対して校正をかけ、ガイドライン順守やスタイルチェックを行う機能をサポートする。

### 原文ボタンでのハイライト

原文ボタンで原文を表示した際にその行やユニットをハイライトしてわかりやすくする

### 同期実行時の現在のファイル同期

sync実行時に現在開いているファイルを展開状態にする

### 複数行のhtmlコメントがある場合の不具合

複数行にわたるhtmlコメントがある場合にhtmlコメント内にmdaitマーカーが振られてしまいネストコメントのようになってしまってエディタ上の表示が崩れる。

### .md以外のファイルのサポート

.md以外のテキストファイル（例: .txt, .json, .yaml .csvなど）もサポートする。
- mdと違いhtmlコメントは使えないためファイル単位で扱う必要がある
- mdaitマーカーはどこに持つのか？検討する必要がある

### frontmatterの翻訳サポート

frontmatter内の値も翻訳できるようにする。
- key: value 形式のうちvalue部分を翻訳する
- どのkeyを翻訳対象とするか設計する必要がある
- frontmatter自体のmdaitマーカーもfrontmatterに持たせ、`mdait.front`とする

### 横並びファイルのtransPairのサポート

現在mdaitはja/enのように言語ごとにディレクトリを分けて翻訳ペアを管理しているが、ja.md/en.mdのように横並びでファイルを配置するケースもある。これらをサポートする。
- 両立は難しそうなので、設定で切り替えられるようにする
