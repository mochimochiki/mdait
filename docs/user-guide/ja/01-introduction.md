# はじめに

## mdaitとは

**mdait**（Markdown AI Translator）は、Markdownドキュメントを**継続的に多言語運用するための**VS Code拡張機能です。一度きりの翻訳ではなく、文書構造に基づいて変更を追跡し、「再翻訳が必要な部分」だけを、用語と文脈を保ったまま継続的にAI翻訳できます。

## 解決する課題

多言語ドキュメントの運用において、以下のような課題がよく発生します：

### 変更箇所の追跡が困難
ドキュメントが更新されても、どの部分を再翻訳すべきかが分かりにくく、手作業での確認が必要になります。

### 翻訳の一貫性維持が難しい
チャットAI翻訳では、翻訳の揺れや再現性の低さが運用上の問題になることがあります。従来のセンテンス単位の機械翻訳では、周辺文脈や文書全体の構造が十分に考慮されず、品質を安定させにくい状況です。

### 用語集が活用されない
用語集を用意していても、実際の翻訳作業に確実に反映・維持する仕組みがなく、用語の統一が困難です。

## mdaitのアプローチ

mdaitは、これらの **「翻訳運用」上の課題を解決するためのツール**です。以下の特徴により、継続的な多言語ドキュメント管理を実現します：

**文書構造に基づく管理**
Markdownドキュメントを見出しレベルに基づいて「ユニット」に自動分割し、それぞれの状態を追跡・可視化します。

**ハッシュベースの変更検出**
各ユニットの内容ハッシュ（CRC32）により、原文と訳文の対応関係を管理し、原文変更を自動検出します。

**文脈を考慮したAI翻訳**
用語集・前後のコンテキスト・前回訳文を含む文脈情報を利用したAI翻訳により、一貫性のある高品質な翻訳を実現します。

**外部ファイル非依存の軽量設計**
管理情報はMarkdownファイル内のHTMLコメントとして埋め込まれ、外部ファイル*や特定のバージョン管理システムに依存しません。
*diff機能のみsnapshotファイルを使用

## 対象ユーザー

mdaitは以下のようなユースケースに最適です：

- **多言語で管理される内部ドキュメント**：企業内の技術文書やマニュアルの多言語管理
- **英語＋ローカル言語を提供するOSSドキュメント**：オープンソースプロジェクトのドキュメント翻訳
- **リリース後も継続的に更新される技術文書**：継続的な更新が必要な製品ドキュメント

## AI利用とデータの扱い

### AI機能
mdaitではLLMを利用し、翻訳や用語検出などの処理を実行します。AIを利用するコマンドのツールチップには`✨[AI]`が表示されます。

### AIなしでの利用も可能
AIを利用しなくとも、ユニット単位の同期や翻訳状態の可視化などの機能で「翻訳運用」上の課題を解決するためのツールとして利用が可能です。

### 対応AIプロバイダー
- VS Code Language Model API（vscode-lm）
- Ollama
- OpenAI API (Not tested yet)

### プライバシー
バックグラウンドでの解析や非公開の通信は行いません。AI APIへの送信内容は、選択されたユニットのテキストと設定された用語集のみです。
